Redesigning Boston Airport Cab's Reservation Webpage

Introduction

Whenever I fly back to Providence, I use Boston Airport Cab to book a car from Boston's Logan Airport to Providence. While I love my experience as a rider, whenever I use the site, the experience is far from seamless. I often find myself being confused and sometimes having to restart the reservation process from start.

So thought it would be a valuable UI/UX task to look at the reservation webpage and...

  1. Analyze its usability
  2. Redesign the webpage with a lo-fi and hi-fi prototype
  3. Implement a responsive redesign

1. Usability Analysis

Using Jakob Nielsen's 10 Usability Heuristics for User Interface Design, I identified usability issues that can be addressed in my redesign.

Heuristic Rating Notes
1: Visibility of system status 1 Nothing indicates status of reservation booking progress
2: Match between system and the real world 2 The reservation booking form is consistent with forms on Airbnb, Expedia and taxi reservation services
3: User control and freedom 1 If a user wants to go back and change reservation properties, all previously selected inputs are cleared. The user has to start over
4: Consistency and standards 3 The same language is used to refer to the properties of booking (e.g. Vehicle, Passengers)
5: Error prevention 3 Errors are indicated if the user tries to continue their reservation
6: Recognition rather than recall 3 Option to start a reservation stands out as it is a separate section on the page in a visual layout that stands out
7: Flexibility and efficiency of use 2 Users can go back and change reservation properties. Users can go straight to the reservation form and start a reservation.
8: Aesthetic and minimalist design 1 There is a lot of irrelevant information that clutters the webpage design. (E.g. images, repeated instructions) There is some information that is better left on the Home and About page e.g. TripAdvisor Reviews. User can get confused and find it hard to focus on the information and actions they need. On smaller devices the user has to scroll until the end to get to reservation form
9: Help users recognize, diagnose, and recover from error 3 There are error messages that tell the user if they've chosen an unsuitable vehicle for the selected number of passengers/luggage.
10: Help and documentation 1 The webpage has both written and video instructions to make a reservation. However, if designed well, instructions are not needed.
